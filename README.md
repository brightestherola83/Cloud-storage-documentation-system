
# Cloud Storage & Documentation System

## Project Overview

This project was created to demonstrate how a school can use cloud storage to organize and manage lecture materials. Amazon S3 was used to store the files, while GitHub was used to document the project and store screenshots and deployment notes.

## Objectives

* Create an Amazon S3 bucket for school lecture materials.
* Organize files into folders.
* Upload lecture and course materials.
* Enable S3 Versioning to keep previous file versions.
* Create a GitHub repository for project documentation.
* Upload screenshots and deployment notes to GitHub.

## Tools Used

* Amazon Web Services (AWS)
* Amazon S3
* GitHub
* Web Browser
* Computer/Laptop

## S3 Folder Structure

The lecture materials were organized into four folders:

* **Lecture-Notes** – for lecture notes and class materials.
* **Assignments** – for assignments and related documents.
* **Past-Questions** – for past examination and practice questions.
* **Course-Materials** – for other course-related materials.

## Versioning

S3 Versioning was enabled on the bucket to help keep previous versions of files when changes are made. This helps protect files from accidental changes or overwriting.

## Documentation

This GitHub repository contains screenshots showing the S3 bucket, folders, uploaded files, and Versioning setup. Deployment notes are also included to explain the steps taken during the project.

## Project Outcome

The project successfully demonstrates how Amazon S3 can be used to store and organize school materials in the cloud. GitHub provides a central place to document the work and keep evidence of the project implementation.
